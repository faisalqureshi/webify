# Webify

## Setting up docker image

Use the provide Dockerfile to construct the webify image.

~~~
FROM ubuntu:16.04

RUN apt-get update
RUN apt-get install -y texlive-full
RUN apt-get upgrade -y
RUN apt-get install -y python-pip python-dev build-essential
RUN pip install --upgrade pip

RUN apt-get install -y haskell-platform
RUN apt-get install  -y pandoc
RUN pip install numpy
RUN pip install pyyaml
RUN pip install pystache
RUN pip install pypandoc
RUN pip install watchdog
~~~

The following command can be used to construct the webify
docker image.

~~~bash
$ docker build -t webify .
~~~

You can confirm that webify image is available as follows

~~~bash
$ docker image ls
~~~

If needed you can remove the image as follows

~~~bash
$ docker rmi webify
~~~

## Using webify docker image to run webify

Use the following command to create an interactive bash shell.  We also mount the three folders needed to run webify.

~~~bash
$ docker run -it -v /path-to/webify:/webify -v /path-to/src:/src -v /path-to/dest:/dest webify
~~~

Check out the `webify-docker-shell-osx.sh` and `webify-docker-shell-windows.bat` files that I use for my windows and osx machines.

## Loading pre-build webify docker image

It is also possible to get the pre-build webify docker image and load it as follows

~~~bash
$ docker load webify-docker-image.tar
~~~

It is possible to create this tar file as follows

~~~bash
$ docker image ls
$ docker save webify > webify-docker-image.tar
~~~
